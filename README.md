# Sistemas Distribuídos - 2025/2026

## Estrutura do Repositório

- **Trabalho Prático 1**: Infraestrutura de monitorização urbana (*One 'Health’*).
  - **Sensor**: Simulação de dispositivos IoT.
  - **Gateway**: Agregação e pré-processamento de dados.
  - **Servidor**: Armazenamento e análise centralizada.

## Convenções e Regras Adotadas

Para manter a integridade do código, seguimos uma convenção de nomes baseada em pastas virtuais:
- `tpX/faseY/docs`: Documentação técnica da fase específica.
- `tpX/faseY/implementacao`: Código fonte e testes de funcionalidade.
> *Nota: O uso da "/" permite uma visualização organizada por pastas nos IDEs (Vs2022, VSCode, Rider, etc.).*

## Stack Tecnológica

- **Linguagem Principal**: C# (.NET 9.0)
- **IDEs**: Projeto Verificado para VSCode, Visual Studio (2022/2026) e Rider. 
- **Protocolos**: TCP/IP (Sockets)
- **Controlo de Versões**: Git com fluxos de trabalho baseados em fases. 
- **Hospedagem do Repositório**: GitHub.

## Como Executar

1. Clonar o repositório;
2. Abrir a solução principal no IDE ou editor de código (VSCode, Visual Studio, Rider, etc.).
3. Para simular a rede, executar o Servidor, a seguir o Gateway e, por fim, as instâncias dos Sensores.
